      Harvey ¶

      Harvey API

      Notes for internal API consumption via Vue.js: Laravel Documentation.

      We make every attempt to adhere to json-api standards.

      Suggestions or Corrections may be directed to engineers@goharvey.com.

      Errors ¶

      Errors Responses ¶

      All Error responses will return an appropriate status code along with a body containing an array of error objects with a key of errors.

      Each error object contains a title, type, and detail.

      {
    "errors": [
        {
            "title": "Unauthenticated.",
            "type": "about:blank",
            "detail": "You do not have access to modify the user with id 1."
        }
    ]
}

      List Users
      GET/api/v1/users

      View a listing of enabled users.

      • Admin required.
      URI Parameters
      HideShow
      per_page
      number (optional) Example: 2

      Enables pagination and set items per page.

      term
      string (optional) Example: john

      Filter results using string against ‘first_name’, ‘last_name’ and ‘email’ values.

      type
      string (optional) Example: patient

      Filter results by user type.

      Choices: patient practitioner admin

      indexed
      boolean (optional) Default: false Example: true

      Use indexed search, faster and close match, intended for “as you type” searches.

      include
      string (optional) Example: practitioner

      Join relationship data.

      Choices: patient practitioner

      Create User
      POST/api/v1/users

      Create a new user. This will also create a related patient.

      URI Parameters
      HideShow
      first_name
      string (required) Example: John

      User First Name, 100 characters max.

      last_name
      string (required) Example: Doe

      User Last Name, 100 characters max.

      email
      string (required) Example: jdoe@somedomain.com

      Valid email, unique, 150 characters max.

      password
      string (required) Example: secret

      6 characters min.

      terms
      boolean (required) Example: true

      Must be true.

      zip
      number (required) Example: 90401

      5 digits, must be a serviceable zip.

      Create an Appointment
      POST/api/v1/appointments

      • No special role required.
      URI Parameters
      HideShow
      appointment_at
      string (required) Example: 2019-03-01 15:30:00

      UTC, format “Y-m-d H:i:s”, must be in the future and before 4 weeks. Practitioner must be available at that time.

      reason_for_visit
      string (required) Example: Why not?

      practitioner_id
      number (optional) Example: 1

      Required if admin or patient. Must be a valid ID.

      patient_id
      number (optional) Example: 1

      Required if admin or practitioner. Must be a valid ID.

      Update Appointment
      PATCH/api/v1/appointments/1

      • Require at least one of these conditions for current logged in user:
        • Admin role
        • Practitioner of the Appointment.
        • Patient of the Appointment, and Appointment status = ‘pending’, and Appointment start_time more than 4 hours from now.
      URI Parameters
      HideShow
      appointment_at
      string (optional) Example: 2019-03-01 15:30:00

      UTC, format “Y-m-d H:i:s”. If present, must be in the future and before 4 weeks. Practitioner must be available at that time.

      reason_for_visit
      string (optional) Example: Why not?

      Reason for visit.

      status
      string (optional) Example: complete

      Appointment status.

      Choices: canceled complete general_conflict no_show_doctor no_show_patient pending
